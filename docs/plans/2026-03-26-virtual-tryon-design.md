# Virtual Try-On Service 설계 문서
## 컬러 샴푸 헤어 프리뷰 + 네일 아트 필름 시뮬레이션

---

## Context

컬러 샴푸와 네일 아트 필름을 제조하는 공장에서, 소비자가 제품 구매 전에 가상으로 체험할 수 있는 서비스를 요청. 목표는 구매 전환율 향상, 브랜드 차별화, 반품/불만 감소.

---

## 핵심 결정사항

| 항목 | 결정 |
|------|------|
| 플랫폼 | 반응형 웹앱 (모바일 + PC + 키오스크) |
| 비즈니스 모델 | B2B2C (공장 → 거래처 브랜드 임베드 → 소비자) |
| 헤어 컬러 | 사진 업로드 + 실시간 AR 카메라 |
| 네일 아트 | 실시간 AR 카메라 |
| 아키텍처 | 서버리스 기본 + 선택적 API (사진 고품질 처리) |
| 제품 규모 | 50개 미만 (소규모) |
| 개발 기간 | 3~6개월 (단계별) |

## 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | Next.js 14 + TailwindCSS | SSG, SEO, 반응형 |
| AR 엔진 | MediaPipe (Face/Hands) + TensorFlow.js | 브라우저 네이티브 AI |
| 헤어 세그멘테이션 (AR) | `emirhanbilgic/hair-segmentation` (TFLite) | 모바일 최적화 |
| 헤어 컬러 (사진) | HairFastGAN (NeurIPS 2024) | 고품질 색상 변환 |
| 네일 세그멘테이션 | MediaPipe Hands + 커스텀 TFLite | 21 랜드마크 기반 |
| 제품 관리 | Headless CMS (Contentful/Notion API) | 비개발자도 관리 가능 |
| 임베드 SDK | Web Components + PostMessage API | 프레임워크 무관 임베드 |
| SNS 공유 | Kakao/LINE/Instagram Share API | 한국 시장 최적화 |
| AI 추천 | 피부톤 HSV 분석 + 룰 기반 매칭 | 단순하고 효과적 |
| 호스팅 | Vercel (프론트) + AWS Lambda (API) | 저비용 |

## 사용자 경험 흐름

### 헤어 컬러 프리뷰
1. 랜딩 → "사진으로 체험" 또는 "실시간 체험" 선택
2. **사진 모드**: 사진 업로드 → 서버 API (HairFastGAN) → 고품질 결과
3. **AR 모드**: 카메라 허용 → 브라우저 AI → 실시간 헤어 컬러 변경
4. 컬러 팔레트에서 제품 색상 선택 (스와이프)
5. 전/후 비교 슬라이더
6. AI 추천: 피부톤 분석 → 잘 어울리는 색상 추천
7. 스크린샷 저장 / SNS 공유 / 구매 링크

### 네일 아트 시뮬레이션
1. 랜딩 → "네일 체험" 선택
2. 카메라 허용 → 손 감지 → 손톱 마스크
3. 네일 필름 디자인 갤러리에서 선택 (좌우 스와이프)
4. 실시간으로 손톱에 필름 디자인 오버레이
5. AI 추천: 피부톤/손 모양에 맞는 디자인 추천
6. 스크린샷 저장 / SNS 공유 / 구매 링크

### B2B 임베드
1. 거래처 관리 페이지에서 브랜드 설정 (로고, 컬러, 제품 선택)
2. 임베드 코드 자동 생성 (`<iframe>` 또는 `<script>` SDK)
3. 거래처가 자사 쇼핑몰에 붙여넣기
4. 소비자가 쇼핑몰에서 바로 체험

## 검증 방법

1. **Phase 1 검증**: 모바일 크롬에서 카메라 허용 후 헤어 컬러 변경과 네일 필름 오버레이가 실시간으로 작동하는지 확인
2. **AR 성능**: 모바일에서 30fps 이상 유지되는지 측정
3. **사진 모드**: 업로드 후 5초 내 고품질 결과 반환 확인
4. **임베드 테스트**: iframe으로 외부 사이트에 삽입 후 정상 동작 확인
5. **크로스 브라우저**: Chrome, Safari (iOS), Samsung Browser 테스트
6. **키오스크**: 풀스크린 모드에서 터치 인터페이스 동작 확인
